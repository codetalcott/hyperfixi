# AI-Powered Analysis Example

This example demonstrates the AI-friendly APIs of the AST Toolkit, including:

- Natural language code explanation
- Intent recognition from user descriptions
- Code generation from natural language
- Quality insights with actionable recommendations

## What This Example Shows

1. **Code Explanation**: Converting complex AST structures into human-readable explanations
2. **Intent Recognition**: Understanding what users want to accomplish from natural language
3. **Code Generation**: Creating hyperscript templates from user intentions
4. **Quality Analysis**: AI-powered insights about code quality and improvements

## Key Features

- **Multi-Audience Explanations**: Tailored for beginners, developers, or experts
- **Pattern Recognition**: Identifying common interaction patterns automatically
- **Template Variations**: Multiple approaches to solve the same problem
- **Actionable Insights**: Specific, implementable improvement suggestions

## Usage

```bash
npx tsx example.ts
```

## Example Interactions

The example demonstrates:

1. **Explaining existing code** in natural language for different audiences
2. **Recognizing user intent** from descriptions like "toggle modal visibility"
3. **Generating code templates** with variations and best practices
4. **Providing quality insights** with automated improvement suggestions

After running this example, you'll understand:

- When to use different explanation detail levels
- How intent recognition can improve developer experience
- How AI-powered insights can guide code quality improvements
