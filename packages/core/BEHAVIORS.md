# Behaviors — moved

> **This document is obsolete.** It described an earlier imperative, class-based
> behavior architecture (factory functions, `destroy()` methods, config interfaces)
> that no longer exists. Behaviors are now defined in **pure hyperscript `source`**,
> compiled on demand — there is no imperative-JS installer path.

The canonical, current documentation lives in the `@hyperfixi/behaviors` package:

- **[packages/behaviors/AUTHORING.md](../behaviors/AUTHORING.md)** — what a behavior
  is (the boundary rule), how to write one, the schema, installing/registering, and
  an LLM-agent checklist.
- **[packages/behaviors/README.md](../behaviors/README.md)** — the curated set, tiers,
  and quick start.
