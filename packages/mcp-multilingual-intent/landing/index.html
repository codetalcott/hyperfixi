Same intent, three languages, identical output

Each input below describes the same SQL query in a different language. The MCP server normalizes all three to the same compiled SQL. This is the core value: prompt_in_japanese === prompt_in_english for behavior.

🇬🇧 English (SVO) BASELINE
select name from users
compile_auto →
{ "domain": "sql", "ok": true, "code": "SELECT name FROM users" }
🇯🇵 Japanese (SOV) IDENTICAL OUTPUT
users から name 選択
compile_auto →
{ "domain": "sql", "ok": true, "code": "SELECT name FROM users" }
🇸🇦 Arabic (VSO, RTL) RTL NATIVE
اختر الاسم من المستخدمين
compile_auto →
{ "domain": "sql", "ok": true, "code": "SELECT الاسم FROM المستخدمين" }

SQL keywords (SELECT, FROM) normalize to standard SQL. Identifiers (column/table names) stay in the input language — Arabic identifiers are preserved, not transliterated. The semantic structure (action + roles) is identical across all three.

Auto-detect across 8 domains

The detect_domain tool classifies natural-language input to the correct domain without a hint. Domain-specific patterns (SQL has FROM/WHERE, BDD has given/when/then) are scored by confidence; the highest match wins.

📊 SQL query sql 0.71
select name from users
detect_domain → sql (confidence: 0.71)
🧪 Behavior spec behaviorspec 1.00
given page /home when click #submit then url contains /success
detect_domain → behaviorspec (confidence: 1.00)

How it works

LLM Agent                    siren-mcp                    Hono Siren API
(Claude, Cursor, etc.)         (bridge)                       (this server)
        |                          |                              |
        |  MCP tools/call          |                              |
        |  action__parse_...  ---->|                              |
        |                          |  POST /api/parse  ---------> |
        |                          |                              |---> @lokascript/framework
        |                          |                              |     DomainRegistry
        |                          |                              |     CrossDomainDispatcher
        |                          |                              |
        |                          |  <--- Siren JSON ----------- |
        |  <--- MCP result --------|                              |
        |                          |                              |
        |  (tools stay stable)     |  (stateless, no rebuild)     |

MCP tools exposed

Tool What it does Required input
action__parse_multilingual_intent Parse text → semantic AST (action + roles + confidence). Auto-detects domain. text
action__detect_domain Classify input to one of 8 domains. Pure classification, no compilation. text
action__compile_auto Detect + compile in one call. Returns SQL, Playwright test, JSX, etc. text
action__get_pattern_examples RAG lookup — few-shot examples from the patterns database for LLM context. prompt
action__list_supported_languages Enumerate all supported language codes. —
