What this demonstrates

  • Fine-grained tracking: the live body reads both globals on its first execution; the runtime records both as dependencies.
  • No wasted re-runs: mutating an unrelated global (try window.hyperfixi.run("set $other to 5") in the console) does not re-execute the body.
  • Multilingual-ready: the body is hyperscript, not JS — it reads naturally and (after Phase 8) will work with localized attribute names too.
