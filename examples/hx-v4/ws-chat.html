WebSocket — ws-connect / ws-send

The form below uses ws-send to forward its fields as a JSON message over the WebSocket opened by ws-connect. Incoming messages tagged with a target field are routed through the existing hx-target/ hx-swap machinery — here the mock echoes each sent message as a swap envelope.

What's happening

  1. The processor sees ws-connect="wss://example/echo" and opens a WebSocket. Submit handlers are installed on every ws-send descendant — here, the form.
  2. On submit, the form's fields are serialized to JSON ({"msg":"..."}) and sent over the socket.
  3. The mock echoes back a swap envelope: {"target":"#messages","swap":"beforeend","data":"<div>..."}. The processor recognizes the envelope shape, runs the swap snippet, and appends the message.
  4. On element removal, MutationObserver closes the socket and removes the form listener.
