SSE — sse-connect / sse-swap

The element below opens an EventSource against /fake-stream and routes incoming tick events into #feed with hx-swap="afterbegin". This page polyfills EventSource with a tiny in-page mock so you can run the demo without a backend.

What's happening

  1. The bundle's htmx-compat processor sees sse-connect="/fake-stream", opens an EventSource, and subscribes a listener for tick events (from the sse-swap value).
  2. Each tick event's data is wrapped in a tiny hyperscript snippet — put <data> at start of #feed — and runs through the same execution path that handles other htmx attributes, honoring hx-target + hx-swap="afterbegin".
  3. On element removal (try devtools → delete node), the MutationObserver closes the EventSource — no leak.
