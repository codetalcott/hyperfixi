What's happening

  1. The hyperfixi-hx-v4.js bundle loads. It bundles the full hyperscript runtime, @hyperfixi/reactivity, and the htmx-compat attribute processor.
  2. Reactivity is installed onto the default runtime at bundle init — that registers the live feature globally and wires $count writes into the notify chain.
  3. On DOMContentLoaded, the htmx-compat scanner sees hx-live, translates it to live put $count into me end, and runs it through the same runtime — so its dep-tracked read of $count subscribes to the reactive graph.
  4. Each button mutates $count via a regular set command. The full runtime's setGlobal() calls notifyGlobalWrite() → reactivity hook fires → the live body re-runs.
