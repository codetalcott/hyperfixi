Verbose event-handler form (now fixed)

This page used to be broken. It demonstrates the verbose-form protocol JSON (kind: "event-handler", nested body:[...]) that the compilation service emitted before the 2026-04-10 fix. The fix landed in <lse-intent>._execute: the element now unwraps event-handler semantic nodes and executes their body commands directly, rather than passing the wrapper to evalLSENode (which would attempt to re-wire a listener at runtime and silently drop the body). See README.md for the investigation.

This is the shape of the old compilation service output for [on event:click body:[toggle patient:.active destination:#sidebar]]. It's still a valid wire format (round-trips through the protocol parser) — it's just more verbose than the new compact form.
