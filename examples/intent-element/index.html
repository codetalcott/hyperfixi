<lse-intent> Demo

Browser custom element that validates LSE protocol JSON and executes it via the hyperfixi runtime.

1. trigger="click" — Toggle .active on #sidebar

No JavaScript wiring. The trigger="click" attribute attaches a click listener automatically; clicking anywhere inside the element fires evalLSENode().

Interaction unavailable
Sidebar content — toggles .active

2. JSON trigger.event sugar — Add .highlighted

No trigger attribute. The element reads the JSON wire-format trigger.event field and auto-wires to that event. This matches the protocol wire format spec and is the most LLM-friendly form.

Interaction unavailable
Target box — gets .highlighted

2b. trigger="manual" — caller invokes refresh()

The element prepares the node but never fires automatically. Useful when you want full control over when execution happens.

Manual-fired box — gets .manual-fired

2c. trigger="submit" — fire on form submit

The element listens on the closest ancestor <form> and calls preventDefault() so the form never navigates.

Submit box — gets .submitted on form submit

2d. trigger="intersect" — fire when scrolled into view

Uses IntersectionObserver. The element fires once when it first scrolls into view (one-shot). Scroll down to trigger.
