Semantic nodes

A semantic node is a language-independent representation of a hyperscript command. It captures the meaning using universal semantic roles from linguistics, rather than the surface syntax of any particular language.

This enables:

  • True multilingual support: parse from any language, render to any language.
  • Meaning preservation: translations maintain semantic equivalence.
  • Runtime optimization: commands can be analyzed at the semantic level.

Node structure

interface SemanticNode {
  kind: 'command' | 'event-handler' | 'conditional' | 'compound' | 'loop';
  action: string;                       // The verb: 'toggle', 'add', 'set', etc.
  roles: Map<SemanticRole, SemanticValue>;
  metadata?: {
    sourceText?: string;
    confidence?: number;
    sourceLanguage?: string;
    patternId?: string;
  };
}

Node types

KindDescriptionExample
commandSingle commandtoggle .active
event-handlerEvent listener with bodyon click toggle .active
conditionalIf/else blockif x show #y else hide #y end
compoundThen-chained commandstoggle .a then put 'x' into #y
loopRepeat/while/for blocksrepeat 3 times toggle .active end

Each role's value is a SemanticValue — a selector, literal, reference (me/you/it), property path, or expression.
